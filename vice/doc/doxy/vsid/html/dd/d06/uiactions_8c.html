Detailed Description

UI actions interface.

System to handle UI actions in an OS/UI-agnostic way.

Used by menu structs, hotkeys and joystick mappings. There will be no Doxygen docblocks for most of the defines, since they're self-explanatory. And obviously I will not bitch about keeping the text within 80 columns here :D

Trigger a UI action.

Calls the action dispatch handler if conditions are met.

If an action is marked as a dialog then it will only be triggered when there is no other dialog active. If an action is marked as blocking i will only be triggered when it isn't marked as busy.
