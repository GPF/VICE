Detailed Description

Global data that custom status bar widgets base their rendering on.

This data is usually set by calls from the emulation core or from other parts of the UI in response to user commands or I/O events.

◆ active_joyports

uint32_t ui_sb_state_s::active_joyports

Which joystick ports are actually available.

This is a bitmask representing notional ports 0-4, which are themselves defined in joyport/joyport.h. Cases like a SIDcard control port on a Plus/4 without other userport control ports mean that the set of active joyports may be discontinuous.

Referenced by ui_statusbar_init(), and ui_update_statusbars().

◆ drives_dual

unsigned int ui_sb_state_s::drives_dual

Drive type changes involving dual-drive status.

Each bit represents a unit, with bit 0-3 -> representing units 8-11. If a bit is set it means the dual-drive state of that unit has changed and the UI needs updating of the widgets for that unit.

Referenced by ui_enable_drive_status().
