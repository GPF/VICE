◆ ARCHDEP_PATH_MAX

#define ARCHDEP_PATH_MAX   PATH_MAX

Arch-independent replacement for PATH_MAX/MAX_PATH.

The maximum size of a pathname.

Note that there are some serious flaws with PATH_MAX and similar constants, so only use this if dynamically allocating memory isn't possible. There also doesn't seem to be consencus on whether PATH_MAX is enough to hold the longest possible path including the terminating NUL character.
