Detailed Description

Extended M3U playlist handling.

Author
Bas Wassink b.was.nosp@m.sink.nosp@m.@zigg.nosp@m.o.nl

Simple API for reading and writing extended M3U playlist files, tailored towards use in VSID.

There is no formal specification of the file format, so information was used from various places on the internet, among which: https://en.wikipedia.org/wiki/M3U

Only a few directives of the extended m3u format are supported. Inline comments are not supported (can't find any reference about whether or not they should be).

Open m3u file for parsing.

Try to open path for reading and set up the parser.

The entry_callback argument is required and will be called by m3u_parse() whenever a normal entry is encountered in path. Its first argument is the entry text and the second argument is the lenght of the entry excluding the terminating nul character.

The directive_callback argument is optional and can be used to handle extended M3U directives. The first argument is the directive ID (

See also
m3u_ext_id_t), the second argument is the text following the directive and the third argument is the length of the text excluding the terminating nul character.

To stop the parser the callbacks must return false.

Parameters
[in]pathpath to m3u file
[in]entry_callbackcallback for normal entries
[in]directive_callbackcallback for extended m3u directives (optional, pass NULL to ignore)
Returns
true on success

◆ playlist_current_dir

char* playlist_current_dir = NULL
static

Current directory for relative entries.

M3UEXT allows using directory-only entries and relative paths, so we keep track of that here.

Initially the directory is set by m3u_open() to the directory of the playlist file being parsed. When a non-file entry is found the contents are either appended to the current dir (if relative) or the current directory is set to the entry (if absolute).
