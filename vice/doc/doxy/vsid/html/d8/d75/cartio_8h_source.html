179/* The following structure is used to register the I/O address range used by a certain device/chip/cartridge.
180 *
181 * The address_mask determines if the defined address range is mirrored across a bigger range, the mask tells
182 * the I/O read/write handler how many bits of the address being accessed are valid for an I/O read or write.
183 *
184 * Some examples:
185 *
186 * start_address | end_address | mask | primary register address | address mirrors
187 * -------------------------------------------------------------------------------
188 * $de00 | $de0f | $00 | $de00 | $de01-$de0f (mirrors of 0xde00)
189 * $de00 | $de03 | $01 | $de00-$de01 | $de02-$de03 ($de02 mirrors $de00 and $de03 mirrors $de01)
190 * $de00 | $deff | $0f | $de00-$de0f | $de10-$deff (15 blocks of 16 bytes mirroring $de00-$de0f)
191 * $de00 | $deff | $7f | $de00-$de7f | $de80-$deff (1 block of 128 bytes mirroring $de00-$de7f)
192 * $de80 | $deff | $7f | $de80-$deff | no mirrors
193 * $de00 | $de0f | $0f | $de00-$de0f | no mirrors
194 * $de00 | $deff | $ff | $de00-$deff | no mirrors
195 *
196 * If the read/write functions of the device ignore the address parameter then it results in the following:
197 *
198 * start_address | end_address | mask | primary register address | address mirrors
199 * -------------------------------------------------------------------------------
200 * $de00 | $deff | $ff | $de00 | $de01-$deff
201 *
202 * Devices can co-exist at the same addresses, but only if they are write only.
203 *
204 * A write to an address that is used by multiple devices just results in the data being written going to all devices.
205 *
206 * A read from an address that is used by multiple devices leads to problems, because all devices will respond with
207 * data to the read, in which case there will be a read-collision. The default action for a read-collision is to report the
208 * user with the names of the devices and the address at which the collision happened, and then detach all devices causing
209 * the read-collision. This default action can be changed to detach only the last inserted device involved in the
210 * read-collision, or AND the returned bytes instead of detaching devices.
211 *
212 * The io_source_prio defines what to do with the returned byte of a device.
213 *
214 * A priority of -1 (IO_PRIO_LOW) means the returned byte of the device should be ignored if other devices return a byte as well,
215 * therefor a device of this priority only returns a byte if it is the only device at the address being read.
216 *
217 * A priority of 1 (IO_PRIO_HIGH) means the returned byte will be returned immediately and no further read-collision checks are done,
218 * therefor a device of this priority will always return a byte, unless it is not the first device in the device-chain at the
219 * address being read. If it is not the first device it will not return a byte, but will also not be part of a read-collision.
220 *
221 * A priority of 0 (IO_PRIO_NORMAL) means there needs to be a check of read-collisions, and the device byte will be used either
222 * in an AND of all returned bytes, or as a device in a read-collision.
223 *
224 * Note: When making emulation code of new devices with their range in the I/O ranges do NOT use IO_PRIO_HIGH unless the device
225 * is a device on the system board, or a cartridge with a passthrough port, if the device doesn't work because of read
226 * collisions do NOT use IO_PRIO_HIGH as a hack to get it to work, the cause is in the emulation code, using IO_PRIO_HIGH
227 * in such a case will only hide the actual problem.
228 *
229 * The I/O ranges that are covered by the I/O system are as follows:
230 *
231 * C64/C128: $d000-$d0ff, $d100-$d1ff, $d200-$d2ff, $d300-$d3ff, $d400-$d4ff, $d500-$d5ff, $d600-$d6ff, $d700-$d7ff,
232 * $de00-$deff, $df00-$dfff
233 *
234 * SCPU64: $d000-$d0ff, $d100-$d1ff, $d400-$d4ff, $d500-$d5ff, $d600-$d6ff, $d700-$d7ff, $de00-$deff, $df00-$dfff
235 *
236 * CBM2: $d800-$d8ff, $d900-$d9ff, $da00-$daff, $db00-$dbff, $dc00-$d9ff, $dd00-$d9ff, $de00-$deff, $df00-$dfff
237 *
238 * PET: $8800-$88ff, $8900-$89ff, $8a00-$8aff, $8b00-$8bff, $8c00-$8cff, $8d00-$8dff, $8e00-$8eff, $8f00-$8fff,
239 * $e900-$e9ff, $ea00-$eaff, $eb00-$ebff, $ec00-$ecff, $ed00-$edff, $ee00-$eeff, $ef00-$efff
240 *
241 * PLUS4: $fd00-$fdff, $fe00-$feff
242 *
243 * VIC20: $9000-$93ff, $9800-$9bff, $9c00-$9fff
244 */
