◆ ui_enable_drive_status()

void ui_enable_drive_status ( ui_drive_enable_t  state,
int *  drive_led_color 
)

Drive related UI.


Drive related UI.

Parameters
stateA bitmask int, where bits 0-3 indicate whether or not drives 8-11 respectively are being emulated carefully enough to provide LED information.
drive_led_colorAn array of size at least NUM_DISK_UNITS that provides information about the LEDs on this drive. An element of this array will only be checked if the corresponding bit in state is 1.
Note
Before calling this function, the drive configuration resources (Drive8Type, Drive9Type, etc) should all be set to the values you wish to display.
Warning
If a drive's LEDs are active when its LED values change, the UI will not reflect the LED type change until the next time the led's values are updated. This should not happen under normal circumstances.
See also
compute_drives_enabled_mask() for how this function determines which drives are truly active
ui_sb_state_s::drive_led_types for the data in each element of drive_led_color

Update hotkey.

The map should contain a valid action ID and the keysym and modmask of the currently registered hotkey for that action. The function is expected to remove the old hotkey from its menu items, if any, set the hotkey for the action to vice_keysym + vice_modmask, and if menu items are present update their accelator labels to the new hotkey.
