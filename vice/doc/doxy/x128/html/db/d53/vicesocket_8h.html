vice_network_socket_address_t * vice_network_address_generate ( const char *  address_string,
unsigned short  port 
)

Generate a socket address.

Initialises a socket address with the value specified as input parameter.

Parameters
address_stringA string describing the address to set.
portA default port number. If address_string does not specify a port number, the number here will be used. If address_string specifies a port number, that one will be used instead.
Returns
socket_address Pointer to a socket address that has been initialised to the specification in address_string (and port, if supported). NULL in case of an error.
Remarks
If address_string starts with a pipe ('|'), then the address_string is treated as a unix domain socket. Otherwise, address_string can be prepended with ip6:// or ip4://, in which case address_string is treated exactly as an IPv6 or IPv4 address, respectively.

If there is no specification, this routine tests for an IPv6 address. If this fails, it tries an IPv4 address.

int vice_network_get_errorcode ( void  )

Get the error of the last socket operation.

This function determines the error code for the last socket operation.

Returns
the error code
Remarks
It does not distinguish between different sockets, thus, make sure to call it directly after an erroneous socket operation. Furthermore, reading the error status does not reset the error code. Thus, do not expect correct results if the previous operation did not produce an error.

Receive data from a connected socket.

This function receives incoming data from a connected socket. opened either by vice_network_client() or vice_network_accept().

Parameters
sockfdThe connected socket to receive from
bufferPointer to the buffer which will hold the received data
buffer_lengthThe length of the buffer pointed to by buffer. This indicates the maximum number of bytes to receive.
flagsFlags for the socket. These flags are architecture dependent.
Returns
the number of bytes received. This can be less than buffer_length.

When the return value is 0, either the other site has gracefully closed the socket (and all data has been received), or no data was received in the case of a non-blocking socket.

In case of an error, -1 is returned.
